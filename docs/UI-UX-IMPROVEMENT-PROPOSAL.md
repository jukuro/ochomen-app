# UI/UX 改善提案（現行アプリ × 再設計指示書）

**参照:** `UI-UX-REDESIGN-BRIEF.md` / `PROGRESS.md`  
**更新:** 2026年6月22日

---

## 現状の位置づけ

| 観点 | 現行アプリ | 指示書の理想 |
|---|---|---|
| プロダクト定義 | おたより・タスク管理（実務寄り） | Family History Platform |
| ビジュアル | Teal / Slate（業務ツール風） | Warm Minimal（#E8826A 等） |
| スキャン体験 | モーダル + スピナー | 3秒撮影 + キャラ演出 + 感情コピー |
| 継続動機 | Paywall・件数制限 | XP / キャラ成長 / ポイント |
| 情報設計 | 5タブ・多機能同居 | 1画面1目的・低密度 |

**結論:** 機能基盤（OCR・カレンダー・お帳面・記録）は指示書の「Capture → Understand → Act → Remember」のうち **Capture〜Act は強い**。**Remember の感情設計と、スキャンの「楽しさ・安心感」が最大ギャップ**。

## Phase 1 — 即効性が高い（既存コードの上に載せる）

### 1-1. デザイントークン移行（Warm Minimal）

**現状:** `teal-600` / `slate-*` が全体に散在。

**提案:**

```css
/* globals.css に CSS 変数を定義し、段階的に置換 */
--color-bg: #FDF9F6;
--color-surface: #FFFFFF;
--color-primary: #E8826A;
--color-secondary: #7BB3A0;
--color-accent: #F5C96A;
--color-text: #3D3535;
--color-muted: #A89F9B;
```

- まず **背景・Primary・CTA** の3点だけ置換（全画面一括はリスク大）
- 角丸を `rounded-xl` → `rounded-2xl`、シャドウを `shadow-sm` に統一
- タッチ領域: ボトムナビ・FAB・主要ボタンを **min-h-[56px] min-w-[56px]** に

**効果:** 「仕事ツール」→「子育て支援ツール」の印象を、機能追加なしで変えられる。

---

### 1-2. 保存完了コピーと micro-celebration

**現状:** Toast「保存しました」、Loader2 スピナー。

**提案:**

| タイミング | 現状 | 改善案 |
|---|---|---|
| OCR中 | スピナー | Skeleton +「プリントを読み解いています」 |
| 保存完了 | 「保存しました」 | **「片付きました ✨」** / **「これで安心です」** |
| 演出 | なし | 0.8秒のチェック + 軽い haptic（PWA） |

実装は `Toast.tsx` と `BatchScanModal` / `ScanModal` の完了ハンドラのみ。Lottie なしでも CSS アニメで十分。

---

### 1-3. ホームを「次の1手」に特化

**現状:** ホームに複数セクション（今日のタスク・記録・FAB 等）。

**提案（1画面1目的）:**

```
┌─────────────────────────┐
│  今夜は、ひとつだけ        │  ← 最上部・最大フォント
│  明日 体操服（次男）       │
│  [済にする]               │
├─────────────────────────┤
│  📷 紙を片付ける          │  ← 親指領域・Primary色・56dp
└─────────────────────────┘
```

- 詳細リストは「もっと見る」で記録/カレンダーへ委譲
- FAB メニューを廃止し **「スキャン」1ボタン** に（考えさせない原則）

---

### 1-4. 検索の前面化

**現状:** ヘッダー内検索アイコン。

**提案:**

- 記録タブ上部に **常時表示の検索バー**（プレースホルダー: 「体操服」「参観日」など）
- カテゴリチップ: 学校 / 保育園 / 医療 / 習い事 / 自治体 / **思い出**（指示書6分類）
- お帳面・日記も検索対象に統合

## Phase 2 — 差別化の核（中規模開発）

### 2-1. スキャンフロー再設計

**現状:** `ScanModal` / `BatchScanModal` — モーダル内カメラ、手動撮影中心。

**指示書との差:**

| 要件 | 現状 | 提案 |
|---|---|---|
| 全画面カメラ | モーダル | 専用 `/scan` ルート or フルスクリーン |
| 自動書類検出 | なし | OpenCV.js / ブラウザ Vision API で輪郭検出 |
| 自動シャッター | なし | 検出安定2秒で「自動で撮影します」→ 撮影 |
| 3秒完了 | 未達 | 起動→検出→撮影を1フローに |

**優先:** まず **自動トリミング + 連続撮影（まとめ撮り）** を全画面化。自動シャッターは Phase 2b。

---

### 2-2. 「紙を食べるキャラクター」MVP

**最小構成（2週間規模）:**

1. ホーム下部 or スキャン完了画面に **固定キャラ（1体）**
2. スキャン1枚 = +10 XP（localStorage）
3. レベルアップ時に短い進化アニメ（CSS scale + 色変化）
4. コピー: **「もう紙は捨てて大丈夫。○○が保管してくれてるよ」**

**データモデル追加案:**

```ts
interface UserProgress {
  totalScans: number;
  xp: number;
  level: number;
  characterId: string;
}
```

**注意:** 子供成長モード（AIキャラ化）は Phase 4。まずは汎用キャラ1体で感情価値を検証。

---

### 2-3. AI解析画面の Skeleton 化

**対象:** `ScanModal`, `BatchScanModal`, `structure-ocr` 待ち時間。

**提案 UI:**

```
┌─────────────────────────┐
│  [キャラクター idle]     │
│  ░░░░░░░  (shimmer)     │
│  ░░░░░░░░░░░            │
│  「プリントを読み解いています」 │
└─────────────────────────┘
```

- `Loader2` を段階的に削除
- 解析完了 → 紙アイコンがキャラ方向へ `translate`（0.5秒）→ 完了コピー

## Phase 3 — プラットフォーム化

### 3-1. タブ構成の再編

**現状:** ホーム / 記録 / お帳面 / 買い物 / カレンダー

**指示書思想に沿った案:**

| タブ | 役割 | 現行との対応 |
|---|---|---|
| きょう | 次の1手 + スキャンCTA | ホーム |
| 思い出 | お帳面・日記・アルバム | 記録 + お帳面 |
| 予定 | カレンダー | カレンダー |
| 買い物 | そのまま | 買い物 |
| わが家 | キャラ・ポイント・設定 | 設定分散を統合 |

「お帳面」を独立タブから **思い出のサブビュー** に統合すると、1画面1目的に近づく。

---

### 3-2. ポイント → ショップ（限定SKU）

**現状:** Stripe Premium のみ。

**提案:**

- スキャン → ポイント（キャラXPと別レーンでも可）
- 交換: シュレッダー・ファイル・オリジナルリフィル等 **文書関連のみ**
- プレミアム: **デジタルブック PDF + 製本注文**（既存「じぃじ・ばぁば製本」プレビューを商品化）

---

### 3-3. デジタルブック（Premium）

**既存資産:** 記録タブの日記、Entry 画像、お帳面セクション。

**提案:**

- 月次自動ドラフト生成（Gemini でキャプション）
- PDF テンプレ（温かい配色 = 指示書パレット）
- 外部印刷 API 連携（製本注文）
